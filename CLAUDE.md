# Проект: Coupang Analytics Dashboard

Next.js 16.2.4 (Turbopack, App Router) + Prisma (SQLite) + Tailwind  
Папка: `C:\Users\PC\Desktop\cpng`  
Dev сервер: `npm run dev` → http://localhost:3000

## Что это за проект

Дашборд для управления магазином на корейском маркетплейсе Coupang.  
Оба товара — **Rocket Growth (RG)**, поэтому везде используется провайдер `rg_open_api`, не `openapi`.

## Чего хочет пользователь

- **Полноценный продукт**, а не заглушки. Если функция делается — делать её полностью, с хорошим UI.
- Все поля, которые можно редактировать через API — добавлять в интерфейс.
- Ничего не скрывать и не упрощать без причины.
- Интерфейс на **русском языке**.

## Критические технические детали

### HMAC подпись (lib/coupang/client.ts)
Знак `?` НЕ включается в подписываемое сообщение:
```
message = datetime + method + urlPath + query  // без '?'
```

### Rocket Growth API
- Заказы: `GET /v2/providers/rg_open_api/apis/api/v1/vendors/{vendorId}/rg/orders`
- Инвентарь: `GET /v2/providers/rg_open_api/apis/api/v1/vendors/{vendorId}/rg/inventory/summaries`
- Товары (чтение/запись): `seller_api` (не rg_open_api)

### Prisma
После изменения схемы **обязательно**:
1. `npx prisma migrate dev --name <name>`
2. `npx prisma generate`
3. Убить Node процесс и `npm run dev` заново (Turbopack кэширует старый клиент)

### params в Next.js 16 (App Router)
В route handlers и page components `params` — это **Promise**:
```typescript
export async function GET(_req: NextRequest, { params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
}
```

### vendorItemId
RG `vendorItemId` ≠ marketplace `vendorItemId`.  
В `products.ts` всегда брать `rocketGrowthItemData.vendorItemId`.  
Цепочка для фото в заказах: `OrderItem.productId` → `Inventory.vendorItemId` → `Product.imageUrl`

## Важные детали реализации

### Купоны в списке заказов
Запросы купонов по orderId делаются **последовательно** (не Promise.all) — параллельные запросы вызывают rate limit у Coupang и часть возвращает пустой массив.

### Korean→Russian перевод статусов
Используется `String.fromCodePoint(0xXXXX, ...)` — строки строятся из hex code points, чтобы избежать проблем с кодировкой файлов при записи корейских литералов.

### RG заказы и адреса
RG orders API не возвращает адрес получателя. При открытии заказа делается попытка получить адрес через стандартный ordersheets API и закэшировать в БД. Если не найдено — блок скрыт.

### Купоны на детальной странице заказа
Дедупликация по couponId — один купон может вернуться несколько раз (по числу товаров в заказе).
