# Страница /discover — Ниши и ресёрч товаров

## Цель

Помочь найти товары без сертификатов для запуска на Coupang: автоматический ежедневный скан ниш + ручная проверка любого товара по ключевому слову с опциональным сравнением фото.

---

## Архитектура

### База данных — новая таблица `NicheOpportunity`

```prisma
model NicheOpportunity {
  id           Int      @id @default(autoincrement())
  keyword      String
  volume       Int
  competition  String   // 높음 / 중간 / 낮음
  verdict      String   // LAUNCH / TEST / AVOID
  trendChange  Int?     // % изменение за 3 мес
  medianPrice  Int?     // ₩
  topKeywords  String   // JSON: [{keyword, volume}]
  scannedAt    DateTime @default(now())
}
```

### Крон-job (instrumentation.ts)

Запускается ежедневно в 7:00 KST. Перебирает ~30 seed-ключей по 6 категориям без сертификатов. Вызывает `quickResearch()` из `lib/naver.ts`. Сохраняет только результаты с вердиктом LAUNCH или TEST. Удаляет записи старше 7 дней.

**Seed-категории (фиксированные в коде):**
- 수납/정리 (хранение): 수납함, 정리함, 서랍정리, 옷걸이, 행거
- 캠핑 (кемпинг): 캠핑의자, 캠핑테이블, 캠핑랜턴, 텐트팩, 캠핑매트
- 자동차용품 (автотовары): 차량용방향제, 차량용수납, 트렁크정리, 주차번호판, 차량청소
- 홈오피스 (домашний офис): 모니터받침대, 키보드받침대, 케이블정리, 독서대, 마우스패드
- 스포츠 (спорт): 요가매트, 폼롤러, 줄넘기, 아령, 운동밴드
- 반려동물 (зоотовары): 강아지장난감, 고양이장난감, 펫빗, 강아지옷, 고양이터널

### Lib — `lib/naver.ts`

Перенести функции из `agent/naver.ts`:
- `fetchKeywordVolumes(keywords)`
- `fetchTrends(keyword)`
- `fetchShoppingResults(keyword, display)`
- `fetchShoppingInsight(keyword, categoryCode)`
- `analyzePrices(items)`
- `analyzeTrends(data)`
- `quickResearch(keyword)` — возвращает `{ keyword, volume, comp, trends, prices, products, verdict, topKws }`

### API маршруты

- `GET /api/discover` — возвращает NicheOpportunity[] за сегодня из БД
- `POST /api/discover/scan` — запускает ручной перескан (для кнопки "Обновить сейчас")
- `POST /api/discover/research` — ручной ресёрч одного ключевого слова, возвращает полный отчёт
- `POST /api/discover/match-image` — Claude Vision анализирует загруженное фото и сравнивает с Naver Shopping результатами по ключу

## UI страницы `/discover`

### Блок 1 — "Находки сегодня"

Шапка: название + время последнего скана + кнопка "Обновить сейчас" (запускает POST /api/discover/scan).

Карточки возможностей (grid), каждая содержит:
- Ключевое слово (корейский)
- Вердикт-бейдж: 🟢 ЗАПУСКАТЬ / 🟡 ТЕСТИРОВАТЬ
- Объём поиска: `45 200 /мес`
- Конкуренция: Низкая / Средняя
- Тренд: `📈 +18%` / `➡️ стабильно` / `📉 -12%`
- Медианная цена: `12 900₩`
- Топ-3 ключа для листинга

Сортировка: сначала 🟢, потом 🟡.

Если скан ещё не запускался сегодня — показывает кнопку "Запустить первый скан".

### Блок 2 — "Проверить товар"

Поле ввода ключевого слова (корейский) + кнопка "Исследовать".

Опциональная загрузка фото (drag-and-drop или клик). Если фото загружено — после ресёрча отправляется в Claude Vision вместе с топ-5 фото конкурентов из Naver Shopping. Claude возвращает: совпадает ли товар с тем что ищут по этому ключу, и объяснение.

Результат ресёрча (~5 сек):
- Вердикт с объяснением причины
- Спарклайн тренда за 12 месяцев (SVG, не recharts)
- Цены конкурентов: мин / медиана / макс + топ-3 популярных диапазона
- Топ-5 ключей для листинга с объёмами и уровнем конкуренции
- Блок рисков (если есть)
- Блок "Анализ фото" (если фото загружено): ✅ совпадает / ⚠️ не совпадает + объяснение Claude

---

## Данные

- Цены берутся из Naver Shopping Search (топ-30 товаров по ключу) — не Coupang, но рынок коррелирует
- Тренды из Naver DataLab Search + Shopping Insight
- Объёмы и конкуренция из Naver Search Ad API
- Анализ фото — Claude API (claude-haiku-4-5 для скорости и экономии)

## Что не входит в скоуп

- Автоматическая отправка в Telegram (это отдельный агент)
- История ресёрчей (не сохраняем ручные запросы в БД)
- Импорт результатов в листинг товара напрямую
